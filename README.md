# Задание 2.1: API tests

Автотесты написаны на стандартной библиотеке Python без внешних зависимостей:
- `unittest` для запуска тестов и ассертов
- `urllib` для HTTP-запросов
- `json`, `re`, `secrets`, `time` для вспомогательной логики

Все файлы решения находятся в корне репозитория, как требуется по условию.

## Что покрыто

Тесты проверяют:
- создание объявления
- получение объявления по `id`
- получение всех объявлений продавца по `sellerId`
- получение статистики по `item id`
- негативные сценарии для невалидного UUID и отсутствующих обязательных полей
- corner cases: повторные GET-запросы и удаление с последующей проверкой чтения
- баги, обнаруженные на стенде и описанные в `BUGS.md`

## Требования

- Python 3.12+ или совместимая версия Python 3
- доступ в интернет до `https://qa-internship.avito.com`

Проверенная версия:

```bash
python3 --version
```

## Запуск

Запуск всех тестов:

```bash
python3 -m unittest -v test_api.py
```

Или так:

```bash
python3 test_api.py
```

## Особенности реализации

- Для каждого теста генерируется уникальный `sellerId` в диапазоне `111111-999999`.
- Для позитивных сценариев создаются уникальные объявления.
- После создания объявления тест пытается удалить его через `DELETE /api/2/item/:id`, чтобы не оставлять тестовые данные.
- Часть тестов намеренно может падать на текущем состоянии стенда: это не ошибка тестов, а воспроизведение дефектов из `BUGS.md`.

## Ожидаемый результат прогона

На корректно работающем сервисе все тесты должны проходить.

На текущем состоянии стенда, проверенном `2026-04-04`, ожидаются падения тестов, которые воспроизводят найденные дефекты:
- некорректный контракт ответа `POST /api/1/item`
- отсутствие валидации отрицательного `price`
- отсутствие валидации отрицательных значений в `statistics`
