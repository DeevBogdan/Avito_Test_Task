# Баги

## BUG-01: `POST /api/1/item` возвращает ответ, не соответствующий контракту из Postman-коллекции

Краткое описание:
После успешного создания объявления сервис возвращает только строковый статус с UUID внутри, вместо объекта созданного объявления.

Шаги воспроизведения:
1. Отправить `POST https://qa-internship.avito.com/api/1/item` с валидным JSON-телом.
2. Проверить тело ответа.

Фактический результат:
Сервис возвращает `200 OK` и тело вида:

```json
{"status":"Сохранили объявление - 3b740fe8-dcec-4d38-8fbb-273cb5a7fe68"}
```

Ожидаемый результат:
Сервис возвращает объект созданного объявления согласно Postman-коллекции, например с полями:
- `id`
- `sellerId`
- `name`
- `price`
- `statistics`
- `createdAt`

Приоритет:
Р2

Окружение:
- Host: `https://qa-internship.avito.com`
- Метод: `POST /api/1/item`
- Дата проверки: `2026-04-04`

## BUG-02: Сервис позволяет создать объявление с отрицательной ценой

Краткое описание:
API принимает отрицательное значение `price` и создаёт объявление.

Шаги воспроизведения:
1. Отправить `POST https://qa-internship.avito.com/api/1/item`.
2. Передать валидное тело, но указать `price = -1`.

Пример тела:

```json
{
  "sellerID": 444444,
  "name": "negative-price-check",
  "price": -1,
  "statistics": {
    "likes": 1,
    "viewCount": 2,
    "contacts": 3
  }
}
```

Фактический результат:
Сервис возвращает `200 OK` и создаёт объявление.

Ожидаемый результат:
Сервис отклоняет запрос с `400 Bad Request`, так как цена не должна быть отрицательной.

Приоритет:
Р1

Окружение:
- Host: `https://qa-internship.avito.com`
- Метод: `POST /api/1/item`
- Дата проверки: `2026-04-04`

## BUG-04: Повторное удаление объявления возвращает противоречивый ответ

Краткое описание:
При повторном `DELETE` одного и того же объявления HTTP-статус и поле `status` в JSON противоречат друг другу.

Шаги воспроизведения:
1. Создать объявление.
2. Выполнить `DELETE /api/2/item/:id`.
3. Повторно выполнить `DELETE /api/2/item/:id` для того же `id`.

Фактический результат:
Сервис отвечает:
- HTTP-статусом `404 Not Found`
- телом `{"result":{"message":"","messages":null},"status":"500"}`

Ожидаемый результат:
Ответ должен быть консистентным:
- либо `404` и тело с `status: "404"`
- либо иной согласованный контракт ошибки

Приоритет:
Р2

Окружение:
- Host: `https://qa-internship.avito.com`
- Метод: `DELETE /api/2/item/:id`
- Дата проверки: `2026-04-04`
